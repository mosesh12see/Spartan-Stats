What is Spartan Stats Dashboard?

The Spartan Stats Dashboard tracks appointment and performance metrics for Spartan operations, providing real-time visibility into call center performance, appointment setting, and team metrics.

‚ö†Ô∏è IMPORTANT: Data Exclusions

The following appointments are EXCLUDED from all statistics:

  • Kentucky (KY) Appointments: All appointments in Kentucky state
  • Missouri (MO) Appointments: All appointments in Missouri state

Reason: KY and MO appointments are tracked separately or excluded from standard Spartan performance metrics based on operational requirements.

Key Metrics Explained

  • Total Appointments: All scheduled appointments (excluding KY and MO)
  • Appointments by State: Geographic breakdown of appointment distribution
  • Call Volume: Number of outbound calls made
  • Contact Rate: Percentage of calls that result in contact with prospect
  • Set Rate: Percentage of contacts that result in appointments
  • Team Performance: Individual and team-level metrics

Calculation Methods

  • Contact Rate: (Total Contacts / Total Calls) √ó 100
  • Set Rate: (Total Appointments Set / Total Contacts) √ó 100
  • Daily/Weekly Totals: Metrics aggregated by date ranges
  • State Filtering: KY and MO excluded during data processing before calculations

Data Source

Data is sourced from call center systems and CRM platforms. The KY and MO exclusion logic is applied during data processing (see generate_spartan_dashboard.py line 231) to ensure accurate representation of target state performance.

Update Schedule

This dashboard is automatically updated hourly from 8am to 8pm via cron job. Metrics reflect real-time or near-real-time data as of the last update.

For detailed technical documentation, see README.md and generate_spartan_dashboard.py (line 231 for exclusion logic) in the project folder.
